# Project Manga-Mon (Working Title)

Project Manga-Mon is a decentralized, Web3-native autonomous sports simulation game built for the Monad blockchain. Players collect, trade, and manage 5v5 football squads consisting of autonomous **AI Persona Agents** minted as dynamic NFTs (dNFTs).

Unlike traditional sports manager games, matches are simulated in real-time, where individual player agents make autonomous tactical decisions driven by their unique personality traits. The core visual hook marries hyper-minimalist 2D pitch gameplay (represented by colored circles) with high-drama, *Blue Lock*-inspired manga panel breakouts showcasing real-time AI internal dialogues generated by the Gemini LLM when critical match events trigger.

---

## 1. Executive Summary & Core Objective

Manga-Mon bridges the data layer of dynamic on-chain NFTs (dNFTs) with an active visual presentation layer. 
- **The Core Hook**: Combining autonomous simulation with dynamic, generative storytelling.
- **Visual Style**: Minimalist 2D pitch circles + high-contrast, stylized monochrome manga comic layouts.
- **Target Network**: Monad (exploiting parallel transaction execution for fast post-match state writes).

---

## 2. Core Feature Set

### 2.1 The Dynamic NFT (dNFT) Player System
Each player character is an on-chain ERC-721 token containing dynamic metadata:
* **Quantitative Stats**: Speed, Passing, Shooting, Defense, Stamina (dynamic range 1-99).
* **Qualitative Persona Traits**: Behavioral modifiers that dictate the agent's decision logic and dialogue.
  * `Arrogant`: Favors shooting and solo dribbling; ignores passing; proud monologues.
  * `Calculative`: Evaluates Expected Value (EV) of plays, making mathematically optimal passes/shots.
  * `Panic-Prone`: Commits high errors when pressed by defenders or low on stamina; frantic thoughts.
  * `Maverick`: High-risk, high-reward behavior; unpredictable dribble drives and long-range shots.
  * `Team-First`: Boosts passing weight; supports teammates; collaborative internal dialogue.

### 2.2 The Autonomous Match Engine (Off-Chain Simulation)
* **Real-time Engine**: Run as a 5v5 2D physics simulation on a Canvas field.
* **Perception Loop**: Every physics tick, players scan positions of the ball, goals, teammates, and opponent defenders.
* **Action Selection**: Players make autonomous choices (Pass, Shoot, Dribble, Intercept) weighted by their quantitative stats and qualitative personality traits.
* **Post-Match Update**: Final match scores, player stats (goals, assists), and stamina degradation are updated back to the Monad contract.

### 2.3 The Gemini-Powered "Manga Panel" UI Engine
The signature aesthetic of Manga-Mon:
* **The Trigger**: High-value shots (Goal Prob > 75%), Low stamina panic states (<20%), or key midfielder assist setups.
* **The Breakout**: The match physics slows down, and a sharp, diagonal comic frame slices onto the screen containing the player's manga artwork/PFP.
* **Dynamic Dialogue Cloud**: A Next.js API endpoint (`/api/dialogue`) calls Google Gemini, feeding the player's profile, the exact match score, time, and current situation to generate a customized, dramatic manga thought bubble (max 15 words) in real-time.

## 5. Development & Run Guide

### Prerequisite Setup
1. Clone your fork and navigate to the project directory:
   ```bash
   cd printf-winners-
   ```
2. Install dependencies:
   ```bash
   npm install
   ```

### Compile & Test Contracts (Foundry)
1. Navigate to the contract folder:
   ```bash
   cd contracts
   ```
2. Build the contract compilation files:
   ```bash
   forge build
   ```
3. Run solidity unit tests:
   ```bash
   forge test
   ```

### Run Frontend & Mock Server (Next.js)
1. Add your Google Gemini API key to your `.env.local` file at the project root:
   ```env
   GEMINI_API_KEY="your_gemini_api_key_here"
   ```
2. Start the development server:
   ```bash
   npm run dev
   ```
3. Open your browser and navigate to `http://localhost:3000` to interact with the wallet manager, player generator, and match simulator.
